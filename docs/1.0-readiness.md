# DavyWhisper 1.0 就绪状态

DavyWhisper `1.0` 是一个稳定的直接下载版 macOS 应用。在 `1.0` 之前，不会添加新的主要产品功能；重点是可靠性、清晰的产品边界、快速升级支持和问题诊断。

## 目标用户

- 需要系统级语音输入并自主选择引擎的 macOS 用户
- 需要在本地或通过 API 转写音频或视频文件的用户
- 需要 Prompt 处理、配置文件和本地自动化的高级用户

## 官方支持的核心功能

- 通过全局热键进行系统级语音输入和文本插入
- 文件转写，包括批量处理和导出
- 使用内置预设和自定义 Action 进行 Prompt 处理
- 基于应用和 URL 的配置文件
- 历史记录、词典和代码片段
- 内置默认集成和默认插件

## 1.0 中的高级功能

以下功能仍是 `1.0` 的一部分，但作为高级或自动化功能定位：

- `/v1/*` 下的本地 HTTP API
- `davywhisper` CLI
- 插件 SDK 和插件清单
- Widget
- 监视文件夹

## 1.0 之前的目标

- 不在 Mac App Store 上线，无 App Store 特定门槛
- 1.0 之前不添加新的主要引擎、插件类或公共 API 系列
- 不对已记录的 HTTP API、CLI 或插件 SDK 做破坏性变更
- `1.0` 无需宽泛的 UI 自动化要求

## 1.x 稳定性契约

### HTTP API

- `/v1/*` 下的已记录端点在 `1.x` 期间保持稳定。
- 在 2.0 之前不得删除响应字段（需先弃用）。
- API 仅限回环地址（loopback），默认禁用，仅供本地自动化使用。

### CLI

- `davywhisper status`
- `davywhisper models`
- `davywhisper transcribe`
- 参数：`--port`、`--json`、`--language`、`--task`、`--translate-to`

### 插件 SDK

- `manifest.json`
- `DavyWhisperPlugin`
- `PostProcessorPlugin`
- `LLMProviderPlugin`
- `TranscriptionEnginePlugin`
- `ActionPlugin`
- `HostServices`

## 发布门槛

只有在满足以下所有条件后才能打 `1.0.0` 标签：

- 应用 scheme 的 `xcodebuild test` 通过。
- `swift test --package-path DavyWhisperPluginSDK` 通过。
- 应用 Release 构建通过。
- 无第一方构建警告。
- 插件清单验证成功。
- README、安全指南和支持矩阵是最新的。
- `1.0.0-rc1` 构建在真实机器上运行多日无 P0/P1 阻断。
- 默认通道保持 `stable`；`release-candidate` 和 `daily` 作为 Sparkle 通道存在，用于预览构建。
- `1.0.0-rc*` 和每日构建作为 GitHub 预发布版本分发，仅在各自通道的共享 Sparkle appcast 中出现，不更新 Homebrew。

## 标记前的手动冒烟检查

- 干净机器上的全新安装
- 权限流程：麦克风、辅助功能、撤销后恢复
- 第一次成功的语音输入
- 文件转写
- 从应用发起 Prompt Action
- 历史编辑/导出
- 配置文件匹配（应用和 URL）
- 插件启用/禁用
- CLI 对本地运行中的服务器
- HTTP API `status`、`models`、`transcribe`
- 从 `0.14.x` 升级，历史记录、配置文件、词典、代码片段、热键和插件状态保留

## 支持与诊断

- 支持请求应始终包含错误日志中 JSON 诊断导出的信息。
- 导出包含应用版本、macOS 版本、API 状态、已激活插件和隐私安全的设置元数据。
- 导出不含 API 密钥、音频数据和转写历史。
